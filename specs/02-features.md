# Feature Specifications

---

## F1: Camera Scanner

The entry point of the app. Works **exactly like Google Translate's camera mode** — point at text, capture the page, see detected text overlaid on the image, and tap to interact.

> **Detailed OCR pipeline spec:** See [07-camera-ocr-overlay.md](07-camera-ocr-overlay.md) for the complete technical specification of the capture-then-OCR pipeline, coordinate mapping, and rendering.

### Behavior

1. App opens directly to a full-screen camera view (live preview, no OCR yet).
2. User taps the **Capture button** (bottom-center) to take a picture.
3. The captured image is displayed full-screen. OCR runs on the high-quality captured image and overlays recognized text — **Google Translate style**: white background fills over detected text regions with the recognized text rendered on top, precisely aligned with the original.
4. User taps to select a **word** from the overlay.
5. Selection triggers the **Explanation Card** (F2).
6. User taps the **Retake button** (same position, now shows refresh icon) to return to live camera preview.

### Controls on Camera Screen

| Element | Position | Purpose |
|---|---|---|
| Source language selector | Top-left | Language of the novel (e.g., Japanese, Korean, French) |
| Target language selector | Top-right | Language for explanations (e.g., English) |
| "Don't Save" toggle | Bottom-left | When ON, lookups are NOT saved to history |
| History button | Bottom-right | Navigate to History screen (F4) |
| Novel selector | Top-center | Select or create the current novel name (for history grouping) |
| Capture button | Bottom-center | Takes a picture and runs OCR on it |
| Retake button | Bottom-center (after capture) | Returns to live camera preview |

### Text Selection

- **Single tap:** Select the tapped word.
- Selected text is highlighted with a white background and black border.

## F2: Explanation Card

A bottom sheet that slides up after selecting a word/phrase.

### Fixed Explanation Categories

The user picks which of these to show (configured in Settings, F5). All selected categories are fetched in a single AI request and displayed together.

| # | Category | Description | Example (word: 墓地 / bochi) |
|---|---|---|---|
| 1 | **Meaning** | Dictionary-style definition in target language | "A graveyard or burial ground" |
| 2 | **Reading / Pronunciation** | How to read the word (furigana, romaji, IPA, etc.) | "ぼち (bochi)" |
| 3 | **Context** | What it means in the context of the surrounding sentence | "Refers to the pet cemetery in the story's setting" |
| 4 | **Examples** | 2–3 example sentences using the word | "古い墓地を訪れた。(I visited an old graveyard.)" |
| 5 | **Breakdown** | Morphological breakdown (kanji meanings, roots, prefixes) | "墓 (grave) + 地 (ground/place)" |
| 6 | **Formality / Register** | Is it casual, formal, literary, slang? | "Neutral/formal. Common in written text." |
| 7 | **Similar Words** | List of 3–5 synonyms or related words (selectable for comparison) | "霊園 (reien), 共同墓地 (kyōdō bochi), 納骨堂 (nōkotsudō)" |

### Similar Words → Comparison Flow

1. The "Similar Words" section shows clickable chips.
2. Tapping a chip opens a **Comparison View** — a side-by-side card showing:
   - Original word vs. Similar word
   - Differences in nuance, usage, formality
   - One example sentence for each to highlight the difference
3. The comparison is generated by a single AI call with a structured prompt.

## F3: "Don't Save" Mode

- A toggle on the camera screen.
- When enabled, lookups are processed normally but NOT written to history.
- Visual indicator: a small crossed-out bookmark icon near the toggle.
- Resets to OFF (saving enabled) each time the app restarts.

## F4: History Screen

Organized by novel tabs.

### Structure

```
┌──────────────────────────────────────────┐
│  [Pet Sematary] [Norwegian Wood] [+ Add] │
├──────────────────────────────────────────┤
│                                          │
│  墓地 (bochi) — graveyard     3/15 2026  │
│  ───────────────────────────             │
│  猫 (neko) — cat              3/14 2026  │
│  ───────────────────────────             │
│  恐怖 (kyōfu) — fear          3/14 2026  │
│                                          │
│  [Search bar at top]                     │
└──────────────────────────────────────────┘
```

### Behavior

- Each novel is a horizontal scrollable tab.
- Words are listed in reverse chronological order (newest first).
- Tapping a word re-opens its Explanation Card with cached data (no re-fetch).
- Search bar filters words within the current novel tab.
- Long-press a word to delete it from history.
- Long-press a novel tab to rename or delete it (with confirmation dialog).
- The `[+ Add]` tab lets the user create a new novel name.

### Novel Management

- Novel is selected/created from the camera screen's novel selector.
- If no novel is selected, words go into a default "Unsorted" tab.
- Novels can be reordered by long-press and drag on tabs.

## F5: Settings Screen

Minimal. Accessed from a gear icon on the History screen.

| Setting | Description | Default |
|---|---|---|
| Explanation categories | Toggle which categories appear on the Explanation Card | All ON |
| Source language | Default source language | Japanese |
| Target language | Default target language | English |
| Clear all history | Delete all saved words and novels | — |
| Export history | Export all history as JSON file | — |
| About | App version, credits | — |
